# @user-interaction-patterns.md - 사용자 인터랙션 절대 규칙

## 🚨 핵심: 절대 헷갈리지 말 것!

## 1. ESC/Interrupt 패턴 🛑
**의미**: "잘못된 방향! 즉시 중단!"
- **즉시 행동**: 모든 작업 중지
- **대기**: 사용자 지시 기다림
- **금지**: 자의적 판단, 계속 진행

## 2. 중간 프롬프트 패턴 💭
**의미**: "작업 중이지만 이것도 기억해"
- **큐잉**: 요청 저장만 하고 작업 계속
- **작업 후 처리**: 완료 후 순차 응답
- **아이디어**: Shrimp 태스크로 추가

## 3. 작업 흐름 규칙
```
현재 작업 진행
    ↓
중간 프롬프트 → [큐에 저장] → 작업 계속
    ↓
작업 완료
    ↓
큐 처리 (설명/태스크 추가)
    ↓
다음 작업
```

## 4. 패턴 구분법
| 신호 | 의미 | 행동 |
|------|------|------|
| ESC 누름 | 중단 명령 | 즉시 정지 |
| 중간 메시지 | 나중 처리 | 큐잉 |
| "계속" | 진행 승인 | 작업 재개 |

## 5. 실수 방지
- ESC를 중간 프롬프트로 착각 ❌
- 중간 프롬프트에 즉시 응답 ❌
- 작업 중 새 작업 시작 ❌
